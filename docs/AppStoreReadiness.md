# App Store readiness

A checklist for shipping String Theory. Anything that needs your Apple Developer account is marked TODO.

## Identity

- App name: String Theory.
- Bundle identifier: `com.javierorraca.stringtheory`. Change it in the StringTheory target's build settings if you need a different one.
- Apple Developer Team: set it in Xcode under the StringTheory target, Signing and Capabilities, with automatic signing.
- Version `MARKETING_VERSION` is 0.1.0 and build `CURRENT_PROJECT_VERSION` is 1. Both live in `project.yml`.

## Signing

- Automatic signing is on. You need a paid Apple Developer account and a Team ID to archive for a device or for TestFlight.
- The simulator build and tests run without signing (`CODE_SIGNING_ALLOWED=NO` in CI).

## Privacy

- The app collects no data, does no tracking, and uses no required-reason APIs.
- The privacy manifest is at `App/StringTheory/Resources/PrivacyInfo.xcprivacy`.
- In App Store Connect, set the privacy label to "Data Not Collected" and keep it in sync with the manifest.
- You still need a privacy policy URL. App Store Connect asks for one even when nothing is collected.

## Assets

- App icon: a generated placeholder at `App/StringTheory/Resources/Assets.xcassets/AppIcon.appiconset/AppIcon.png`. TODO: replace with final art. Regenerate the placeholder with `swift tools/make-app-icon.swift <path>`.
- Launch screen: generated by Xcode from the Info.plist key `INFOPLIST_KEY_UILaunchScreen_Generation`. TODO: design a real one if you want more than a blank screen.
- Fonts: system faces for now. TODO: bundle Space Grotesk, Hanken Grotesk, and JetBrains Mono (all SIL OFL) and register them in the Info.plist.

## Device and capabilities

- Deployment target: iOS 17.0.
- Device family: iPhone, portrait only (set in `project.yml`). TODO: decide on iPad and landscape support.
- No special required device capabilities. The app only needs audio output, which every device has.

## TestFlight

1. Set your Team ID and a real bundle id in `project.yml`, then run `xcodegen generate`.
2. In Xcode, select a generic iOS device, then Product > Archive.
3. In the Organizer, distribute the archive to App Store Connect.
4. In App Store Connect, add the build to TestFlight, fill in the test details, and invite testers.

## Before you submit

- Screenshots for each required device size.
- App description, keywords, and a support URL.
- Privacy policy URL.
- Age rating questionnaire.
- Confirm the build runs on a real device, not only the simulator.
